Portfolio

I started by designing a personal brand in order to create a style that represented myself and were consistent with my personality.

Brand Personality

The first exercise I did to design my portfolio was Jennifer Aaker's The five dimensions of brand personality. The goal is to pick 2 adjectives that represent your brand and 1 that doesn't. In my case:

I'm charming and organized but no rigid.

Brand Affinity

The second exercise I did was a brand affinity. With this exercise we can define our brand personality by referencing people, things and even other brands that we think have a similar personality. So, if I was a famous character I'd be Mónica Geller, if I was an animal I'd be a bengal cat, and if I was another brand I'd be Muji.

Moodboard

Once my brand personality and affinity were defined, I created a moodboard with images where those ideas were represented.

Then, I did a desirability test with my colleagues at IronHack in order to confirm my moodboard and my brand attributed were consistent. Some of the adjectives they used to describe the moodboard were: organized, ordered, nordic, cozy, hygge and relaxed.

Style tile

I got my brand's color palette from the moodboard's images, and with that I created the style tile. I chose Open Sans as typography, a Sans serif with open and neutral forms, with a friendly appearance and great legibility.

Error Screens

I designed two error screens in Sketch to get inspired for the mood I wanted my portfolio to portray.

Logo

For the logo, I used Adobe Illustrator to trace a personal image and create my logo from it. Here you can see how I traced the image's contour and then filled it with color.

Portfolio Design

The last step, before starting to code, was designing the layout using Sketch. I exported the design to Zeplin once it was finished, so it was easier to work on the coding side.

HTML and CSS

We learned the basics of HTML and CSS during the 7th week of the UX/UI Bottcamp at Ironhack. In that week we coded our own personal portfolios using tools like Zeplin, iTerm, Visual Studio Code and Github. The week was a busy one, but we learned a ton of things. It was also very satisfying seeing the final result. Here's the Github's project.
